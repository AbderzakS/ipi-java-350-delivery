Introduction

 Maintenant que nous avons développé notre application web et qu'elle tourne en local, il faut avoir un hébergement permettant le déploiement de l'application afin de la rendre accessible à d'autres utilisateurs.
  • Les pré-requis avant de réfléchir à l'hébergement
  • Les différents types d'hébergement
  • Choisir le bon hébergement
  • La maintenance
 Ces éléments sont à prendre en compte au plus tôt dans le processus de développement car ils peuvent potentiellement influer sur des choix organisationnels ou technologiques.

Les pré-requis pour un déploiement serein

  • Une bonne connaissance des contraintes et enjeux de l'application
  • Un système de gestion des versions
  • Un système de packaging
  • Un système de déploiement (automatisé ?)
  • Un moyen centralisé et sécurisé permettant de documenter les processus de déploiement.

Une bonne connaissance des contraintes et enjeux de l'application

 Il est nécessaire de connaître la réponse aux questions suivantes :
  • Quelles sont les contraintes de sécurité de l'application ?
  • Quelles sont les contraintes réglementaires de l'application ?
  • Quel est le budget réservé à l'hébergement ?
  • Quelles sont les ressources humaines et matérielles disponibles en interne ?
  • Quels environnements sont nécessaires pour le client ?
  • ...
 Pensez-vous à d'autres questions ?

Un système de gestion de version

 Il est très important d'avoir un système de gestion des versions clair et carré afin de faciliter le développement et la maintenance de l'application.
 Une bonne maîtrise du SCM est requise ainsi que le respect du workflow par l'ensemble des développeurs

Un système de packaging

 Un système de packaging permet de gérer de manière simple et centralisée les artefacts de votre application (JAR, ZIP...) dans le but de les déployer dans un environnement.
 Il fonctionne de pair avec le système de gestion de version et est parfois couplé à un repository central public ou privé (comme NPM pour les projets NodeJS ou MVN pour les projets Java par exemple).

Un système de déploiement (automatisé ?)

 Il faut pouvoir déployer de manière fiable et efficace n'importe quelle version de votre application sur n'importe quel environnement en ayant le moins d'impact possible sur les utilisateurs.
 Idéalement ce déploiement est automatisé ou a minima documenté de manière très précise.

L'hébergement

 Il existe de nombreuses possibilités pour héberger une application web. Il faut d'abord définir s'il doit être interne ou externe.
  • Hébergement interne
  • Confidentialité
  • Sécurité
  • Maîtrise
  • Coûts
  • Performances
  • Responsabilités
  • Infrastructures
  • Hébergement externe
  • Flexibilité
  • Responsabilités
  • Coûts
  • Performances
  • Maîtrise
  • Responsabilités
  • Confidentialité
 Ce choix peut être influencé par des contraintes réglementaires ou tout simplement matérielles

Les différents types d'hébergement

 Une fois le choix interne/externe effectué, il reste encore beaucoup de possibilités.
  • Hébergement mutualisé
  • Hébergement dédié
  • Hébergement dédié virtualisé
  • Hébergement cloud
